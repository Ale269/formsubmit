The project

My role:

I independently took care of all the steps necessary to carry out the project, from start to finish.

Duration:

From August 15th to october 10th, 2021.

What we have to know

The Goals

The project aims to help anyone find work, especially people with disabilities.
It's a pro-bono project.

Pain Points and User Research

I created a questionnaire for a group of possible users, after identifying the best candidates I continued with interviews in order to understand the problems that could be solved. This is what i found:

Point #1

Users would like to understand well in advance the tasks they will have to perform.

Point #2

Users would like to be able to expand their contacts with companies.

Point #3

Users spend a lot of time looking for a job.

Point #4

Users wish they could have a greater variety of choices.

Point #5

Users would like to be able to get in touch with companies.

Point #6

Users would like to be able to post ads to recruit staff.

Competitor Analysis

What are the solutions that companies facing the same problems have found? How do they relate to users? Let's do a thorough general analysis.

The data

I analyzed: the company's target, the website on mobile and desktop, the features, accessibility, user flow, navigation, graphic elements, corporate tone and contents, strengths points and weak points.

What i found

The target was heterogeneous, the websites analyzed were all responsive, the tone was concise and informal, the graphic elements were simple and intuitive, there were generally few features, the user flow was direct and linear. Thanks to this analysis I created the next steps.

Digital wireframes and Prototyping

Later I created some digital wireframes and a prototype.

User testing

I did some unmoderated user studies.
Five participants were involved, the duration of the test was about 10 minutes, each of them was asked to carry out some activities, explaining aloud their reasoning and what they thought.

Objective

The goal was to understand if users found the user flow intuitive and if they were able to send an application.
I also wanted to test the layout, see if there were any errors or problems that I had not taken into consideration.

Key performance indicator

the values that were taken into consideration were:

#1 Time on task

#2 User error rates

#3 Conversion rates

#4 Net promoter score

#5 System usability scale

What i found out

From the usability tests I received some feedback, the most frequent ones were:

#1

Users wanted filters to improve their search.

#2

Users wanted an option to find similar jobs.

#3

Some users wanted the option of being able to attach a file in the application.

#4

Some buttons had to be moved and enlarged to facilitate user flow.

Iteration

After the usability tests I corrected the errors I found.
I repeated the usability tests a few times to fix a few small details.
