The project

My role:

I independently took care of all the steps necessary to carry out the project, from start to finish.

Duration:

From June 30th to August 10th, 2021.

What we have to know

The Goals

The aim of the project was to design a responsive website that could help users in the ticket ordering process, while also allowing them to monitor available seats.

Pain Points and User Research

I created a questionnaire for a group of possible users, after identifying the best candidates I continued with interviews in order to understand the problems that could be solved. This is what i found:

Point #1

Users want to be able to choose their seats.

Point #2

Users don't want to spend too much time in line.

Point #3

Users want to know how many seats are available before going to the cinema.

Point #4

Users may have problems with the language.

Point #5

Users want to be warned if vacancies are running out.

Competitor Analysis

I analyzed the solutions adopted by the main competitors in the field of cinema to get a general idea of what could be the best strategies.

The data

I analyzed: the company's target, the website on mobile and desktop, the features, accessibility, user flow, navigation, graphic elements, corporate tone and contents, strengths points and weak points.

What i found

The target was heterogeneous, the platforms were all responsive, the tone was informal, fun and friendly, the graphics were generally simple but attractive, the website usually had an intuitive layout, not many features were present.
Thanks to this analysis I was able to plan the next steps..

Digital wireframes and Prototyping

User testing

I did some unmoderated user studies.
Five participants were involved, the duration of the test was about 10 minutes, each of them was asked to carry out some activities, explaining aloud their reasoning and what they thought.

Objective

The aim of the study was to determine if users were able to buy a movie ticket and find the related QR code to show later.

Key performance indicator

the values that were taken into consideration were:

#1 Time on task

#2 User error rates

#3 Conversion rates

#4 Net promoter score

#5 System usability scale

What i found out

From the usability tests I received some feedback, the most frequent ones were:

#1

Users wanted more payment options.

#2

Users wanted more visible details about the selected movie.

#3

Some users wanted an option to save their purchase data so they don't have to re-enter them in the future.

#4

A button has been added to call up a chat bot to help users in case of need.

Iteration

After the usability tests I corrected the errors I found.
I repeated the usability tests a few times to fix a few small details.
